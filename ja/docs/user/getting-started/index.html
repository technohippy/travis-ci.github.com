Travis CI

オープンソースコミュニティのための分散ビルドシステム。

はじめに

プロジェクトがビルドできることを確かめる

  • Rubyプロジェクト: RubyプロジェクトをTravis CIでビルドするには、リポジトリにデフォルトタスクをtestタスクにしたRakefileを用意して下さい。それだけです。Travis CIはデフォルトでは、もしGemfileがあれば、まずbundle installを実行し、それからrakeを実行します。ビルドが成功したことはコマンドのレスポンスコードによって判断されます。
  • Erlangプロジェクト: ErlangプロジェクトをTravis CIでビルドするには、Rebarを利用するか、testsというMakefileターゲットが必要です。ビルドの成功はコンパイルステップのレスポンスコードとeunitテストによって判断されます。Erlangプロジェクトについての詳細

もし必要ならデータ接続を含むあらゆる側面を設定できます。

サインイン

Travis CIの利用を始めるには、Github OAuthを使ってサインインする必要があります。Travis CIに行き、sign inリンクをクリックして下さい。

Githubは読み書きのアクセス権限を与えるように要求します。Travis CIはあなたのリクエストに応じてリポジトリにサービスフックを設定するために書き込み権限を必要としますが、それ以外の部分には一切触れません。

サービスフックの追加

サインインすると自分のプロフィールページに移動します。自分のリポジトリの一覧が表示されるでしょう。Travis CIでフックしたいリポジトリのそれぞれについてスイッチのon/offを切り替えて下さい。

それからそのプロジェクトのGithubサービスフックページを開き、もし自動的に入力されていなければ、Travisサービス用の設定にGithubユーザー名とTravisトークンを貼りつけて下さい。

ビルドプロセスをトリガーするものは何か？

ビルドを開始するには、リポジトリに何かをコミットしてプッシュしても構いませんし、Guthubのサービスフックページに移動して、Travisのための“Test Hook”ボタンを利用することもできます。

これによりビルドジョブがTravis CIのジョブキューに入り、ワーカーが利用可能になるとすぐにビルドが実行されます。

ビルドの設定を調整する

リポジトリのルートに.travis.ymlファイルを追加するとビルドを構成できます。詳細についてはビルドの設定を参照して下さい。

データベース、利用可能なインフラストラクチャなど

Travisのワーカーは、MySQL、PostgreSQL、MongoDB、memcachedなど、あなたが依存している大体のサービスとうまく動きます。

テストスートが使用するデータベース接続をどのように設定すればいいかを理解するにはデータベースの設定を参照して下さい。
